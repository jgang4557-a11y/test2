# VTuber 프로필 페이지 제작 — 전역 지침

지한(프리랜서 웹 제작자)의 VTuber/스트리머(SOOP Live) **프로필 페이지** 제작.
스택: 바닐라 HTML/CSS/JS + Supabase + Cloudflare Pages(GitHub 연동).
스트리머마다 **별도 레포 · 별도 Supabase 프로젝트.** "공통 템플릿 + 스트리머별 값" 구조.

---

## 🚫 자주 틀리는 3가지 — 이거 어기면 처음부터 다시

**1. 레퍼런스 = 그 화면 그대로. 크기·비율·가로폭·세로 길이까지.**
- 이미지 받으면 그 프레임 1:1. 가로형 한 화면이면 한 화면으로 끝 — **짧은 게 정답.**
- `--page-width:1100` 강제 금지. 레퍼런스 폭/비율에 맞춰. "베끼지 마" 주석 무시.
- 픽셀 자신 없으면 추측으로 그리지 말고, **쓸 치수(가로·세로·비율)를 먼저 한 줄로 적고** 시작해.

**2. 안 시킨 건 절대 추가하지 마.**
- 레퍼런스/지시에 **없는** 섹션·카드·VOD·링크·푸터·배지 추가 금지.
- "기본이니까" "있으면 좋으니까" 금지. **멈추고, 묻지도 말고, 그냥 안 한다.**
- 모자라 보여도 추가하지 마 — 부족하면 사용자가 말한다.

**3. 레이아웃 겹침·깨짐 금지 (제일 흔한 사고).**
- **메인 만들 때 스켈레톤 `index.html`의 기존 더미 섹션을 전부 지우고 새로 짠다. 그 위에 덧붙이지 마라 — 옛 마크업 + 새 마크업이 겹쳐서 깨진다.**
- **한 파일에 전체 레이아웃은 딱 하나.** 여러 시안을 한 화면에 겹쳐 쌓지 마.
- 레이아웃을 `position:absolute`로 잡지 마 → **flex/grid + 정상 문서 흐름.** (absolute는 장식 요소만)
- 탭 비교가 필요하면 비활성 탭은 **반드시 `display:none`** — 전부 떠서 겹치게 두지 마.
- 다 만들면 **섹션끼리 겹침·overflow 삐져나옴 없는지 확인**하고 마무리.

## ★ 핵심 (가이드 요약)

- **메인 ≠ 프로필.** 둘 다 풀. 메인 = 한 화면 큐레이션 **대시보드**(히어로 + 프로필카드 + 이번주 일정 + About + 핵심 링크를 카드로 꽉). 프로필 = 더 깊은 **전체 상세**. 메인이 히어로만 띡·밑이 텅 = ❌.
- **화면 품질(공통):** 떠있는 흰 시트 + 양옆 여백 5~8% + 카드 밀도(빈 뷰포트 금지). 풀블리드 금지. 레퍼런스가 휑해도 이 품질은 항상 입힌다.
- **메인 모드:** 레퍼런스 받음 → 재현(위 🚫1). 안 받음 → 디자인 모드(새 레이아웃 + 탭 비교 쇼케이스 3~4개, 각 탭 풀 대시보드). "여러 개·탭 쇼케이스·베끼지 마"는 전부 메인+디자인 모드 한정.
- **프로필은 모드 무관 항상 풀콘텐츠.** 프로필정보·TMI·좋아/싫어·일정·VOD·링크 + 보이스 5종에서 골라 섞기. 정돈된 한 레이아웃, 탭 쇼케이스 안 만듦. 줄이는 건 콕 집어 요청할 때만.
- **프로필·카테고리 = 확정된 메인 디자인 상속.** 표준 템플릿 없음. 빌드 순서: **메인 확정 → 프로필 → 나머지 카테고리.**
- **비-프로필 페이지·admin·overlay = 제공 템플릿 그대로 복사.** 새로 코딩 금지. 바꾸는 건 색·폰트·로고·마스코트·분류 이름·`{{}}`값·fx.js 모양 4줄뿐.
- **옷장 = 포스터+기존 토글(`dress/index.html`).** 옛 3:4 그리드 금지.
- **fx.js는 오버레이 뺀 모든 페이지** `</body>` 위에. 메인=`fx.js`, 서브=`../fx.js`.
- **테마 변경은 테마만.** admin 로직·DB 배선·기능 JS 절대 안 건드림.
- **이미지=링크.** 핫링크(옷장/공지)는 `<img referrerpolicy="no-referrer">`. SQL은 `supabase-schema.sql` 한 번에 Run.

## 일하는 방식

- **한국어 비격식·간결.** 서론·되묻기·다중 질문 싫어함. 짧게 줘도 의도 파악해서 바로 실행.
- "예시/시안"엔 되묻지 말고 바로. 레퍼런스 주면 되묻지 말고 1:1.
- 다단계: 스코프만 한 번 확인 → 내부 배치 처리 → **최종 결과물만** 전달(중간 산출 X).
- 셋업 안내(GitHub 시크릿 등)는 한 스텝씩, 확인 받고 다음.
- **파일명 영문만.** 산출물은 작업 폴더에.

## 알려진 함정

- 확정된 레이아웃은 고정 — 테마 변형은 스타일만, 구조 안 바꿈.
- fx.js 폴백 입자는 `.fxfb` 전용 클래스로만 (`#fx span` 포괄 선택자 금지 — 배포본 입자 깨짐). `typeof window.fxHearts==='function'` 가드.
- **FOUC 방지:** 본문 `opacity:0` → 데이터 적용 후 `body.ready` 페이드인 + 타임아웃 폴백(메인 1600ms / 서브 1200ms).
- 빌드 스크립트를 import해서 실행 금지(출력 폴더 리셋·패치 되돌림).
- Supabase 무료 티어 7일 무활동 시 일시정지 → keep-alive Action(월/목).
- GitHub 웹 에디터는 YAML 들여쓰기 망침 → 단일 라인 또는 plain text 경유 붙여넣기.
